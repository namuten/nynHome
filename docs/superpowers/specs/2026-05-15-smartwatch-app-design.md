# CrocHub 스마트워치 앱 설계

**날짜:** 2026-05-15  
**작성:** Claude (기획/설계)  
**구현 담당:** Gemini  
**상태:** 설계 완료, 구현 대기

---

## 개요

CrocHub 운영자(사이트 주인)가 스마트워치에서 중요 알림을 확인하고 방문자와 빠르게 소통할 수 있는 워치 앱.

**목표:**
- 손목에서 중요 알림(댓글, 방명록) 즉시 확인
- 사전 설정 문구로 빠른 답변 전송
- 폰을 꺼내지 않아도 소통 가능

**플랫폼 순서:** Wear OS (안드로이드) 먼저 → 완성 후 watchOS (Apple Watch)로 포팅

---

## 아키텍처

```
CrocHub 백엔드 (기존)
    │ FCM 푸시 (중요 알림 태그 추가)
    ▼
CrocHub 안드로이드 앱 (기존 앱에 Wear OS 기능 추가)
    │ Wearable Data Layer API (MessageClient)
    ▼
Wear OS 앱 (신규 개발)
    │ 퀵 액션 탭
    ▼
CrocHub 안드로이드 앱  ──► CrocHub 백엔드 API
```

**핵심 원칙:**
- 별도 컴패니언 앱 설치 없음 — 기존 CrocHub 안드로이드 앱에 통합
- 워치는 항상 폰을 통해 백엔드와 통신 (Standalone 방식 아님)
- 기존 FCM 인프라 최대한 재활용

## 알림 범위

### 워치로 전달하는 알림 (중요 알림만)
| 이벤트 | 설명 |
|--------|------|
| 새 방명록 등록 | 방문자가 방명록에 글을 남긴 경우 |
| 새 댓글 | 내 콘텐츠에 댓글이 달린 경우 |
| 댓글 답글 | 기존 댓글에 답글이 달린 경우 |

### 워치로 전달하지 않는 알림
- 조회수 급증, 시스템 알림 등 — 폰에서만 확인

---

## 퀵 액션

알림 상세에서 선택 가능한 액션:

| 액션 | 동작 |
|------|------|
| 좋아요 | 댓글 또는 방명록에 하트 반응 |
| 빠른 답변 | 사전 설정 문구 중 선택 후 즉시 전송 |
| 읽음 처리 | 알림 닫기 |

**빠른 답변 문구:** 관리자 웹 대시보드에서 커스터마이징 가능 (3~5개). 기본 예시:
- "고마워요!"
- "곧 답변할게요 😊"
- "감사합니다 🐊"
- "방문해줘서 고마워요!"

> 긴 텍스트 타이핑은 워치에서 지원하지 않음 — 사전 문구 선택 방식만 사용.

## Wear OS 앱 화면 구성

### 화면 1: 워치페이스 Complication
- 미확인 알림 수 표시 (숫자 뱃지)
- 탭하면 앱 진입

### 화면 2: 알림 목록
- 최근 중요 알림 최대 10개 스크롤 목록
- 각 항목: 아이콘 (댓글/방명록 구분) + 보낸 사람 이름 + 경과 시간
- 탭하면 상세 화면으로 이동

### 화면 3: 알림 상세
- 내용 전체 텍스트 (스크롤 지원)
- 하단: 퀵 액션 버튼들 (좋아요 / 빠른 답변 / 읽음 처리)

### 화면 4: 빠른 답변 선택
- 사전 설정 문구 리스트 표시
- 선택 즉시 전송
- 전송 완료 시 햅틱 피드백 + 확인 화면

## 기술 스택

### Wear OS 앱 (신규)
- **언어:** Kotlin
- **UI:** Jetpack Compose for Wear OS
- **통신:** Wearable Data Layer API (MessageClient / DataClient)

### 기존 안드로이드 앱 (추가)
- `play-services-wearable` 의존성 추가
- `WearableListenerService` 등록
- FCM 수신 후 워치 전달 로직 추가

### Apple Watch 포팅 시 (추후)
- 안드로이드 앱 → iOS 앱 (WatchConnectivity 프레임워크)
- Wear OS 앱 → watchOS 앱 (SwiftUI)
- 구조 동일, 플랫폼 API만 교체

## 백엔드 변경사항

기존 코드 최대한 유지, 아래만 추가:

### 1. FCM 알림 필터 태그
FCM 전송 시 중요 알림에 `priority: "watch"` 태그 추가. 안드로이드 앱이 이 태그를 보고 워치 전달 여부 결정.

### 2. 빠른 답변 문구 API
```
GET  /api/admin/watch/quick-replies     — 설정된 문구 목록 조회
PUT  /api/admin/watch/quick-replies     — 문구 수정/저장
```

### 3. 워치 퀵 액션 API
기존 API 재활용:
- 좋아요: 기존 댓글 좋아요 API
- 답변 전송: 기존 댓글 작성 API

## 구현 순서 (Gemini 참고용)

1. **백엔드** — FCM 태그 추가 + 빠른 답변 문구 API
2. **안드로이드 앱** — Wearable 의존성 추가 + WearableListenerService 구현
3. **Wear OS 앱** — 화면 4개 구현 (목록 → 상세 → 퀵 액션 → 빠른 답변)
4. **관리자 웹** — 빠른 답변 문구 설정 UI 추가
5. **통합 테스트** — 폰 ↔ 워치 연동 확인

## 로컬 테스트 환경 (에뮬레이터)

Android Studio에서 실제 기기 없이 에뮬레이터로 개발 및 테스트 가능.

### 환경 구성

1. **Wear OS AVD 생성**
   - Android Studio > Device Manager > Add Device
   - Wear OS Small Round 또는 Large Round 선택
   - API Level 30 이상 권장

2. **폰 에뮬레이터 페어링**
   - Wearable Data Layer 테스트를 위해 안드로이드 폰 에뮬레이터도 함께 실행
   - **Google Play가 활성화된 에뮬레이터** 이미지여야 페어링 가능 (Play Store 아이콘 있는 것)
   - `adb` 를 통해 폰-워치 에뮬레이터 간 페어링 수행

3. **페어링 명령 (참고)**
   ```bash
   # 폰 에뮬레이터 포트 확인 후 워치와 연결
   adb -s emulator-5554 forward tcp:5601 tcp:5601
   adb -s emulator-5556 connect localhost:5601
   ```

### 에뮬레이터 vs 실제 기기

| 항목 | 에뮬레이터 | 실제 기기 |
|------|-----------|---------|
| UI 개발/확인 | 충분 | 좋음 |
| 폰 ↔ 워치 통신 | 가끔 불안정 | 안정적 |
| 햅틱 피드백 | 미지원 | 지원 |
| 최종 검증 | 비권장 | 권장 |

**결론:** UI 개발은 에뮬레이터로, 알림 전달·퀵 액션 통신 테스트는 실제 기기에서 최종 확인 권장.
